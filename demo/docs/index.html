Meridian Web SDK Documentation

Overview

You will need to obtain a read-only API token from the Meridian Editor before you can use the Meridian Web SDK. 


    MeridianSDK.init({
      api: MeridianSDK.createAPI({
        environment: "production",
        token: "{{ INSERT_YOUR_READONLY_TOKEN_HERE }}"
      })
    });

    const element = document.getElementById("meridian-map");
    const map = MeridianSDK.createMap(element, {
      locationID: "{{ INSERT_YOUR_LOCATION_ID_HERE }}",
      floorID: "{{ INSERT_YOUR_FLOOR_ID_HERE }}",
      placemarks: { all: true },
      tags: { all: false }
    });

    // Later on...
    map.update({
      placemarks: { all: false },
      tags: { all: true }
    });

You can obtain the location ID and floor ID for a given map by navigating to it in the editor and grabbing the IDs from the URL in your browser.

MeridianSDK.init(options)

  • options: an object with any of the following keys:
    • api: the object returned from MeridianSDK.createAPI

If all your Meridian SDK calls will be for the same organization, you should call init and save your API object here so you don't have to pass it to every createMap call later in your app.

If you are using separate organizations on the same page, you can manually store the createMap calls in variables and pass them as api properties to createMap elsewhere in your code. 


    MeridianSDK.init({
      api: MeridianSDK.createAPI({
        environment: "production",
        token: "{{ INSERT_YOUR_READONLY_TOKEN_HERE }}"
      })
    });

Returns an API object with the following properties:

    const element = document.getElementById("meridian-map");
    const map = MeridianSDK.createMap(element, {
      locationID: "{{ INSERT_YOUR_LOCATION_ID_HERE }}",
      floorID: "{{ INSERT_YOUR_FLOOR_ID_HERE }}",

      map.update({
        tags: { all: true },
        placemarks: { all: false }

MeridianSDK.createAPI(options)

  • options: an object with any of the following keys:
    • environment: defaults to "production"; valid values are any of the following: "production"
    • token: a read-only API token obtained from the Meridian Editor

Returns an API object with the following properties:

  • token: the API token
  • environment: the API environment
  • axios: an instance of axios configured with a base URL for the current environment and including an HTTP header with the token
  • openStream(options): a function to open a streaming connection for real time tag updates 

    const api = MeridianSDK.createAPI({
      environment: "production",
      token: "{{ INSERT_YOUR_READONLY_TOKEN_HERE }}"
    });

api.openStream(options)

  • options: an object with any of the following keys:
    • locationID: the location ID
    • floorID: the floor ID
    • onInitialTags: an optional callback function called with an array of tag data after the stream connection is successful
    • onTagUpdate: an optional callback function called with a single tag data when that tag reports its position
    • onTagDisappear: an optional callback function called with a single tag data when that tag exits its current floor
    • onClose: an optional callback function called when the stream is closed
    • onException: an optional callback function called when the stream reports an error

Returns an API object with the following properties:

  • close(): a method to close the stream 

    const connection = api.openStream({
      locationID: locationID,
      floorID: floorID,
      onInitialTags: tagArray => {
        console.log(tagArray);
      },
      onTagDisappear: tag => {
        console.log(tag);
      },
      onTagUpdate: tag => {
        console.log(tag);
      },
      onClose: () => {
        console.log("Closing!");
      },
      onException: err => {
        console.log("An error occurred:", err)
      }
    });

    // Close connection after 10 seconds
    setTimeout(() => {
      connection.close();
    }, 10000);
